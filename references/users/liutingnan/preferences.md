# Preferences: liutingnan

This file records one example user-level presentation overlay.

These are not core protocol rules.
They are overlay preferences that shape how outputs are written and arranged.

## 1. Language defaults

- default to Chinese prose
- keep proper nouns, technical terms, and stable labels in English when that improves precision
- prefer Chinese-first section headings

## 2. Readability defaults

- human-readable first
- prefer a quick-read summary layer before the heavy dossier body
- in `positioning`, prefer:
  - one-sentence overview early
  - key differences early
  - clusters and bridge roles before appendix-style detail
- avoid outputs that read primarily like schema dumps for downstream AI use

## 3. Summary-density defaults

- keep `full-profile` as a summary-on-top heavy dossier
- summaries may compress, but body sections should remain analytically dense
- in `positioning`, prefer “一句话总览 + 主簇 / 桥梁 / 分工” before deeper support
- in `roadmap`, give one primary route first, not many equal routes

## 4. Evidence-boundary presentation defaults

- preserve evidence and coverage boundaries
- prefer moving detailed boundary discussion toward later sections or appendix layers when readability benefits
- keep `Claims intentionally not made`, `Minimal Source Notes`, or equivalent restraint signals available when needed
- do not let readability hide thin evidence

## Use rule

Use this overlay for:

- language choice
- reading order
- summary density
- evidence-boundary placement

Do not use it to override:

- router logic
- evidence protocol
- coverage protocol
- artifact model
- domain taxonomy
