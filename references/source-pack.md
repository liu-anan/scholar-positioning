# Source Pack

Use this reference when running `full-profile` from scratch.

The goal of the source-pack is to stabilize evidence before prose generation.

## What a source-pack is

A `source-pack` is a compact structured note that sits between raw public retrieval and final scholar dossier writing.

It is not the final profile.
It is the evidence staging layer that helps the model:

- avoid jumping from a few URLs to a full dossier too early
- separate strong evidence from weak inference
- reconstruct trajectory and neighbors from source patterns
- make later updates cheaper

Read these files together:

## Minimum discipline

A source-pack should make these things explicit before prose writing:

- identity confidence
- source coverage
- representative works spread
- phase clues
- method clues
- neighbor clues
- claims to avoid

If these are not visible in the pack, the later profile will usually compensate with improvised prose.

## Writing principle

The source-pack is allowed to be compact.
The profile is not.

Use the pack to hold evidence in compressed form, then write the profile as human-facing prose.
Do not leave the profile sounding like the pack.

## Core handoff

After the source-pack is filled and passes the checklist:

- use `Timeline Clues` to draft `研究轨迹`
- use `Genealogy / Context Clues` to draft `学术谱系`
- use `Method Clues` to draft `方法论印章`
- use `Neighbor Clues` to draft `竞争格局`
- use `Coverage Notes` to draft `证据边界`

The point is not to slow the workflow down.
The point is to stop unstable evidence from turning into polished but weak profile prose.
