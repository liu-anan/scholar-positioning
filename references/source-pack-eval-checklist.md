# Source-Pack Eval Checklist

Run this checklist before turning a `source-pack` into a `full-profile`.

This is a pre-drafting gate, not a user-facing section.

## Pass / Warn / Stop

Use three outcomes:

- `PASS`: good enough to draft with current confidence level
- `WARN`: draft is allowed, but specific sections must be downgraded or marked provisional
- `STOP`: keep retrieving; do not draft a confident profile yet

## Identity gate

- `PASS` canonical scholar identity is stable
- `PASS` defining affiliation or equivalent field anchor is known
- `WARN` aliases remain partial but homonym risk is low
- `STOP` homonym risk is still unresolved

## Source-family gate

- `PASS` at least two source families were recovered when reasonably available
- `PASS` at least one work-metadata source is present
- `WARN` identity/context sources are thin but work inventory is strong
- `STOP` the pack relies on one thin page plus a few titles

## Representative-works gate

- `PASS` at least five representative works when available
- `PASS` works span more than one phase
- `WARN` only three or four works are recoverable, but they still show route change
- `STOP` all selected works come from one phase and no broader inventory exists

## Trajectory gate

- `PASS` phase clues come from problem/method/application change
- `PASS` at least three temporal buckets are visible unless the scholar is genuinely sparse
- `WARN` one phase boundary is provisional
- `STOP` phases are inferred only from institution chronology

## Genealogy gate

- `PASS` at least two relation categories are supported
- `WARN` mentor chain is unclear, but training environment and theory community are visible
- `STOP` there is no usable context for how the scholar entered the field

## Method gate

- `PASS` a recurring method combination is visible
- `PASS` likely formation period is visible
- `WARN` method boundary conditions are thin
- `STOP` the method section would collapse into a slogan only

## Neighbor gate

- `PASS` direct neighbors and adjacent routes are separated
- `WARN` neighbor evidence is mostly venue/topic adjacency, so landscape must be marked provisional
- `STOP` direct-neighbor claims would rely on speculation alone

## Coverage gate

- `PASS` coverage level can be named clearly as `C1` or `C2`
- `WARN` coverage is `C2` with visible weak spots
- `STOP` effective coverage is `C3 sparse abstract-only` and the draft would still make strong benchmark or bridge claims

## Dossier-readiness gate

- `PASS` the pack can already support all six prompt question groups:
  - `学术谱系`
  - `研究轨迹`
  - `方法论印章`
  - `核心发现实质分析`
  - `学者类型定位`
  - `竞争格局`
- `WARN` one group is still thin, but the draft can mark it provisional without collapsing the dossier
- `STOP` two or more groups are still too thin and the draft would mostly rely on clean summary prose

Thin means one of these is true:

- the section would collapse into a slogan
- the section would repeat generic bio wording
- the section would mostly rest on one unsupported inference
- the section would lose phase logic or relation logic if written honestly

## Drafting rules after the gate

If the pack is `PASS`:

- draft the profile normally
- keep coverage limits visible in `证据边界`

If the pack is `WARN`:

- draft the profile
- weaken `学术谱系`, `竞争格局`, or `基本信息` as needed
- add explicit avoided claims
- state which prompt group remained thin

If the pack is `STOP`:

- keep searching
- or produce only a clearly downgraded partial draft
- do not treat it as stable map input
- do not label it as a finished heavy dossier

## Common reasons for false confidence

- clean prose hiding thin retrieval
- citation-rich middle phase masking missing early formation
- neighbor lists copied from intuition rather than evidence
- biography fragments being mistaken for genealogy

If one of these is true, downgrade the draft even if the pack feels coherent.
