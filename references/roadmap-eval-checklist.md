# Roadmap Eval Checklist

Run this checklist after drafting a `roadmap`.

Keep it short.
Its job is to catch silent drift from `positioning` into unsupported forecasting.

## Pass / Warn / Stop

- `PASS`: roadmap stays within upstream evidence and is operationally useful
- `WARN`: roadmap is usable, but some route claims or priorities are too strong
- `STOP`: roadmap has drifted into unsupported field forecasting or invented structure

## Trigger gate

- `PASS` the user explicitly asked for a roadmap or next-step route
- `STOP` roadmap was generated only because a positioning map existed

## Input gate

- `PASS` roadmap names the upstream positioning artifact
- `PASS` roadmap names the upstream profiles
- `STOP` roadmap is effectively built from memory, impression, or a thin scholar list

## Route gate

- `PASS` one primary route is clear
- `WARN` two secondary routes exist but remain clearly lower priority
- `STOP` the draft presents many equal-priority routes

## Evidence inheritance gate

- `PASS` route claims do not exceed the strongest critical upstream link
- `WARN` one sequencing choice is plausible but still only `E3`
- `STOP` `E3` bridge claims are written as settled strategy

## Forecast drift gate

- `PASS` the roadmap is framed as benchmark sequencing, transfer route, or补档优先级
- `WARN` one paragraph drifts toward bigger field claims but is explicitly downgraded
- `STOP` the roadmap reads like a full field future or inevitable technical trajectory

## Missing-middle gate

- `PASS` the draft states which missing scholar, method, or evidence gap would most likely flip the route judgment
- `WARN` this is mentioned only briefly
- `STOP` the roadmap acts complete despite obvious corpus gaps

## Stop-line gate

- `PASS` the draft explicitly says what it does not claim
- `PASS` the draft includes a stopping line for stronger conclusions
- `STOP` no stopping line exists

## Good final test

Ask:

- if one key missing scholar were added tomorrow, could the route order change?

If yes, the roadmap must say so.
If it does not, downgrade it.
